# Secure File Manager 本地部署指南

欢迎参与本项目的开发！为了确保大家能够顺利地在本地运行和调试，请按照以下步骤进行环境配置和项目启动。

## 1. 运行环境要求

在开始之前，请确保你的电脑上已经安装了以下软件：
- **JDK**: 17 及以上版本
- **Maven**: 3.6+
- **Node.js**: v16+ (建议 v18+ 或 v20+)
- **MySQL**: 8.0+

## 2. 数据库配置与初始化

1. 启动你的 MySQL 服务，连接到数据库控制台（可以使用 Navicat、DataGrip 或命令行）。
2. 创建一个新的数据库，命名为 `secure_file_manager`：
   ```sql
   CREATE DATABASE secure_file_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 在该数据库中执行项目 `database/` 目录下的 SQL 脚本以初始化表结构：
   - 使用工具或命令行直接执行 `database/init.sql`（该文件已包含完整的表结构、索引和所有新增字段，只需执行一次即可）

## 3. 后端服务部署

1. **导入项目**：使用 IntelliJ IDEA (或 VS Code) 打开项目的 `backend` 文件夹。
2. **下载依赖**：等待 Maven 自动同步下载相关依赖包（或者手动执行 `mvn clean install`）。
3. **修改配置文件**：
   打开 `backend/src/main/resources/application.yml`，找到并修改以下关键配置以适配你本地的环境：
   
   - **数据库连接** (根据你的 MySQL 用户名密码修改):
     ```yaml
     spring:
       datasource:
         username: root
         password: 你的本地数据库密码
     ```
   - **文件存储路径** (重要：修改为你本地实际的目录路径确保有读写权限):
     ```yaml
     secure-file:
       storage-root: D:/你的本地路径/SecFileManager/storage
     ```
   
4. **启动项目**：找到 Spring Boot 的启动类，点击 Run 启动服务。成功启动后，后端将在 `http://localhost:8080/api` 运行。

## 4. 前端服务部署

1. **进入前端目录**：打开终端并跳转到 `frontend` 目录：
   ```bash
   cd frontend
   ```
2. **安装依赖**：建议使用 `npm` 或 `pnpm` 安装项目依赖：
   ```bash
   npm install
   # 或者 pnpm install
   ```
3. **启动开发服务器**：
   ```bash
   npm run dev
   # 或者 pnpm dev
   ```
4. 控制台会输出类似 `http://localhost:5173/` 的访问地址，在浏览器中打开该地址即可访问前端界面，此时应该能正常调用跑起来的后端接口了。

## 5. 常见问题排查 (FAQ)

- **前端无法请求后端 / CORS 跨域问题**：确保 Vite 的 proxy 已经配置正确（`frontend/vite.config.js`）且后端监听了 8080 端口。
- **文件上传失败**：检查 `application.yml` 中的 `storage-root` 目录是否存在且有写入权限。
- **数据库连接失败**：仔细检查 `application.yml` 中 MySQL 的 Url、账号、密码及服务是否正常运行。

## 6. 开发环境模拟生产部署（kkFileView）

本项目在最终服务器部署时，将使用 `kkFileView` 作为系统的核心高级文件访问引擎。为了能在本地开发中完美模拟线上服务器的真实环境体验，我们需要在本地部署一个 `kkFileView` 测试容器进行联调。

### 6.1 本地启动测试版 kkFileView

你有两种方式在本地启动 kkFileView 进行联调：

**方式一：IDEA 源码启动（免装 Docker）**
如果你不想在本地安装配置 Docker 环境，组长已经将 kkFileView 的可用版本上传到了项目的源码仓库中。
1. 更新/拉取最新代码，使用 IntelliJ IDEA 直接打开其中包含的 kkFileView 工程目录。
2. 打开其配置文件（通常是 `application.properties`），确保找到并配置信任白名单 `trust.host = *`（或把本机的 IP 地址加入白名单中），这一步防止本地请求被拒绝。
3. 找到主启动类运行即可（服务将默认启动在 `8012` 端口）。
4. 启动完成后，同样在浏览器访问 `http://127.0.0.1:8012` 测试是否就绪。

**方式二：使用 Docker 容器启动**
如果你习惯容器化或者本机已经装好了 Docker，这也是一种非常干净的环境隔离首选：
1. **拉取并运行镜像**：
   打开终端输入以下命令（通过环境变量传入白名单域名配置）：
   ```bash
   docker run -dit -e KK_TRUST_HOST=* -p 8012:8012 --name kkfileview keking/kkfileview
   ```
2. **验证部署**：
   启动成功后，浏览器中访问 `http://127.0.0.1:8012`。如果能看到 kkFileView 的首页，说明本地联调引擎已就绪。

### 6.2 本地联调配置（后端）

请在 `backend/src/main/resources/application.yml` 中配置预览地址，使其指向刚才搭建好的本地测试容器：

```yaml
secure-file:
   viewer:
      url-template: http://127.0.0.1:8012/onlinePreview?url={base64Url}
```

**联调机制说明：**
- `{base64Url}` 后端接口会自动将其转换为携带 `&fullfilename=xxx` 参数的 Base64 安全回调地址，从而与 kkFileView 服务无缝对接。
- **全量预览路由**：系统前端（`FilePreview.vue`）将会默认对所有访问的文件优先请求该本地模拟器进行加载。
- **容错自动降级**：即便本地联调时遇到 kkFileView 对某个文件格式不支持或报错 404，前端也能静默侦测并回退到原生基础组件渲染（如图库、代码着色等），确保开发调试过程不受阻断。

### 6.3 本地联调坑点防范（前端 IP 访问避坑）

**非常重要！** 
当你通过 Docker 运行 kkFileView 时，如果浏览器地址栏仍然采用 `localhost` 访问前端系统，那么后端最终传给容器的下载地址也会变成 `http://localhost:8080/api/...`。kkFileView 位于容器内，它的 `localhost` 会指向容器它自己，导致拉取文件失败报「404」或者「不受信任的主机」的错误。

**解决办法**：
1. **修改前端配置暴露 IP**：在 `frontend/vite.config.js` 的 `server` 配置中添加 `host: '0.0.0.0'`，使前端可以通过真实 IP 访问。
   ```javascript
   export default defineConfig({
     // ...
     server: {
       host: '0.0.0.0',
       port: 5173,
     },
   });
   ```
2. **使用真实 IP 访问本系统**：重启前端后，不要再用 `localhost`，一定要在浏览器上输入你本机的**局域网真实 IP** 来访问系统前端。例如：`http://192.168.x.x:5173`。

这样，发送给 kkFileView 的文件回源链接就会携带真实的内网 IP，保证处于 Docker 容器中的它能正常获取宿主机的文件流进行预览。
