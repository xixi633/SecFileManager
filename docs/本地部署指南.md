# Secure File Manager 本地部署指南

本文档只保留本地开发必须步骤，去掉了过时配置和冗余说明。

## 1. 运行环境要求

- JDK 17+
- Maven 3.6+
- Node.js 16+（建议 18+ 或 20+）
- MySQL 8.0+

## 2. 数据库初始化

1. 创建数据库：

```sql
CREATE DATABASE secure_file_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行初始化脚本：
- `database/init.sql`

## 3. 后端启动

1. 打开 `backend` 工程。
2. 下载依赖：

```bash
mvn clean install
```

3. 修改 `backend/src/main/resources/application.yml` 的本地配置：
- `spring.datasource.username`
- `spring.datasource.password`
- `secure-file.storage-root`（必须是本机可读写目录）

4. 启动 Spring Boot，默认地址：
- `http://localhost:8080/api`

## 4. 前端启动

1. 进入前端目录：

```bash
cd frontend
```

2. 安装依赖：

```bash
npm install
```

3. 启动开发服务：

```bash
npm run dev
```

4. 浏览器访问：
- `http://localhost:5173/`

## 5. kkFileView 本地联调（可选）

不需要 kkFileView 也能完成大部分开发；仅在需要文档类在线预览时启用。

### 5.1 启动 kkFileView（Docker 示例）

```bash
docker run -dit -e KK_TRUST_HOST=* -p 8012:8012 --name kkfileview keking/kkfileview
```

启动后访问：
- `http://127.0.0.1:8012`

### 5.2 后端预览配置（当前默认推荐）

`backend/src/main/resources/application.yml`：

```yaml
secure-file:
  viewer:
    url-template: "{scheme}://{host}:8012/onlinePreview?url={base64Url}"
    source-base-url: "auto"
```

说明：
- `url-template` 使用动态占位符，避免写死 IP。
- `source-base-url: auto` 表示后端按当前请求自动生成回源地址。

### 5.3 Docker 场景注意事项

如果 kkFileView 在 Docker 容器中，而后端回源地址生成成了 `localhost`，容器内可能无法访问宿主机后端。可改为显式地址：

```yaml
secure-file:
  viewer:
    # kkFileView 的地址（按你的实际部署修改）
    url-template: "http://127.0.0.1:8012/onlinePreview?url={base64Url}"
    # 容器可访问的后端地址（二选一）
    source-base-url: "http://host.docker.internal:8080/api"
    # 或者 source-base-url: "http://你的局域网IP:8080/api"
```

修改配置后需要重启后端。

## 6. 常见问题

- 前端请求失败：确认后端已启动，且前端地址可访问。
- 文件上传失败：确认 `secure-file.storage-root` 目录存在且有写权限。
- 数据库连接失败：检查 MySQL 用户名、密码、端口、库名。
- kkFileView 预览失败：
  - 先确认 `http://127.0.0.1:8012` 可访问。
  - 再检查 `secure-file.viewer.url-template` 和 `source-base-url` 是否与当前环境一致。
