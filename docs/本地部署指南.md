# Secure File Manager 本地部署指南

欢迎参与本项目的开发！为了确保大家能够顺利地在本地运行和调试，请按照以下步骤进行环境配置和项目启动。

## 1. 运行环境要求

在开始之前，请确保你的电脑上已经安装了以下软件：
- **JDK**: 17 及以上版本
- **Maven**: 3.6+
- **Node.js**: v16+ (建议 v18+ 或 v20+)
- **MySQL**: 8.0+

## 2. 数据库配置与初始化

1. 启动你的 MySQL 服务，连接到数据库控制台（可以使用 Navicat、DataGrip 或命令行）。
2. 创建一个新的数据库，命名为 `secure_file_manager`：
   ```sql
   CREATE DATABASE secure_file_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. 在该数据库中执行项目 `database/` 目录下的 SQL 脚本以初始化表结构：
   - 使用工具或命令行直接执行 `database/init.sql`（该文件已包含完整的表结构、索引和所有新增字段，只需执行一次即可）

## 3. 后端服务部署

1. **导入项目**：使用 IntelliJ IDEA (或 VS Code) 打开项目的 `backend` 文件夹。
2. **下载依赖**：等待 Maven 自动同步下载相关依赖包（或者手动执行 `mvn clean install`）。
3. **修改配置文件**：
   打开 `backend/src/main/resources/application.yml`，找到并修改以下关键配置以适配你本地的环境：
   
   - **数据库连接** (根据你的 MySQL 用户名密码修改):
     ```yaml
     spring:
       datasource:
         username: root
         password: 你的本地数据库密码
     ```
   - **文件存储路径** (重要：修改为你本地实际的目录路径确保有读写权限):
     ```yaml
     secure-file:
       storage-root: D:/你的本地路径/SecFileManager/storage
     ```
   
4. **启动项目**：找到 Spring Boot 的启动类，点击 Run 启动服务。成功启动后，后端将在 `http://localhost:8080/api` 运行。

## 4. 前端服务部署

1. **进入前端目录**：打开终端并跳转到 `frontend` 目录：
   ```bash
   cd frontend
   ```
2. **安装依赖**：建议使用 `npm` 或 `pnpm` 安装项目依赖：
   ```bash
   npm install
   # 或者 pnpm install
   ```
3. **启动开发服务器**：
   ```bash
   npm run dev
   # 或者 pnpm dev
   ```
4. 控制台会输出类似 `http://localhost:5173/` 的访问地址，在浏览器中打开该地址即可访问前端界面，此时应该能正常调用跑起来的后端接口了。

## 5. 常见问题排查 (FAQ)

- **前端无法请求后端 / CORS 跨域问题**：确保 Vite 的 proxy 已经配置正确（`frontend/vite.config.js`）且后端监听了 8080 端口。
- **文件上传失败**：检查 `application.yml` 中的 `storage-root` 目录是否存在且有写入权限。
- **数据库连接失败**：仔细检查 `application.yml` 中 MySQL 的 Url、账号、密码及服务是否正常运行。

如果在本地部署中遇到其他未涉及的疑难问题，请在群里交流。祝开发愉快！
