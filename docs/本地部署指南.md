# Secure File Manager 本地部署指南

本文档只保留本地开发必须步骤，去掉了过时配置和冗余说明。

## 1. 运行环境要求

- JDK 17+
- Maven 3.6+
- Node.js 16+（建议 18+ 或 20+）
- MySQL 8.0+

## 2. 数据库初始化

1. 创建数据库：

```sql
CREATE DATABASE secure_file_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行初始化脚本：
- `database/init.sql`

## 3. 后端启动

1. 打开 `backend` 工程。
2. 下载依赖：

```bash
mvn clean install
```

3. 修改 `backend/src/main/resources/application.yml` 的本地配置：
- `spring.datasource.username`
- `spring.datasource.password`
- `secure-file.storage-root`（必须是本机可读写目录）

4. 启动 Spring Boot，默认地址：
- `http://localhost:8080/api`

## 4. 前端启动

1. 进入前端目录：

```bash
cd frontend
```

2. 安装依赖：

```bash
npm install
```

3. 启动开发服务：

```bash
npm run dev
```

4. 浏览器访问：
- `http://localhost:5173/`

## 5. kkFileView 本地联调（可选）

不需要 kkFileView 也能完成大部分开发；仅在需要文档类在线预览时启用。

### 5.1 启动 kkFileView（Docker 示例）

```bash
docker run -dit -e KK_TRUST_HOST=* -p 8012:8012 --name kkfileview keking/kkfileview
```

启动后访问：
- `http://127.0.0.1:8012`

### 5.2 后端预览配置（当前默认推荐）

`backend/src/main/resources/application.yml`：

```yaml
secure-file:
  viewer:
    url-template: "${SECURE_FILE_VIEWER_URL_TEMPLATE:{scheme}://{host}:8012/onlinePreview?url={base64Url}}"
    source-base-url: "${SECURE_FILE_VIEWER_SOURCE_BASE_URL:auto}"
    browser-source-base-url: "${SECURE_FILE_VIEWER_BROWSER_SOURCE_BASE_URL:auto}"
    browser-unreachable-hosts: "${SECURE_FILE_VIEWER_BROWSER_UNREACHABLE_HOSTS:host.docker.internal,docker.for.mac.host.internal,docker.for.win.host.internal}"
```

说明：
- `url-template` 使用动态占位符，避免写死 IP。
- `source-base-url`：kkFileView 服务端回源地址（文档类转换链路）。
- `browser-source-base-url`：浏览器直接拉流地址（视频/音频链路）。
- `browser-unreachable-hosts`：浏览器不可直连主机名单，命中后自动回退到当前请求域名。

### 5.3 Docker 场景注意事项

如果 kkFileView 在 Docker 容器中，而后端回源地址生成成了 `localhost`，容器内可能无法访问宿主机后端。可改为显式地址：

```yaml
secure-file:
  viewer:
    # kkFileView 对外地址（按你的部署修改）
    url-template: "http://127.0.0.1:8012/onlinePreview?url={base64Url}"
    # 文档转换回源：保证 kkFileView 容器可访问
    source-base-url: "http://host.docker.internal:8080/api"
    # 媒体回源：保证浏览器可访问（常用 localhost/域名）
    browser-source-base-url: "http://localhost:8080/api"
```

### 5.4 推荐的环境变量注入方式

```bash
# kkFileView地址（可选）
SECURE_FILE_VIEWER_URL_TEMPLATE=http://127.0.0.1:8012/onlinePreview?url={base64Url}
# 文档回源（kkFileView容器可达）
SECURE_FILE_VIEWER_SOURCE_BASE_URL=http://host.docker.internal:8080/api
# 媒体回源（浏览器可达）
SECURE_FILE_VIEWER_BROWSER_SOURCE_BASE_URL=http://localhost:8080/api
```

修改配置后需要重启后端。

## 6. 常见问题

- 前端请求失败：确认后端已启动，且前端地址可访问。
- 文件上传失败：确认 `secure-file.storage-root` 目录存在且有写权限。
- 数据库连接失败：检查 MySQL 用户名、密码、端口、库名。
- kkFileView 预览失败：
  - 先确认 `http://127.0.0.1:8012` 可访问。
  - 再检查 `secure-file.viewer.url-template` 和 `source-base-url` 是否与当前环境一致。

## 7. 聊天模块联调（一期）

当前版本已支持：
- 好友申请/通过/拒绝
- 好友一对一会话
- 文本消息（WebSocket 实时推送）
- 文件消息（复用文件库上传 + 授权下载）

### 7.1 新增配置项

`backend/src/main/resources/application.yml`：

```yaml
secure-file:
  chat:
    file-share-expire-hours: 24
    message-page-size: 20
```

说明：
- `file-share-expire-hours`：聊天文件分享授权有效期（小时）。
- `message-page-size`：聊天消息默认分页大小。

### 7.2 WebSocket 握手说明

- 后端握手端点：`/api/chat/ws`
- 鉴权方式：`token` 查询参数（JWT）

示例：

```text
ws://localhost:8080/api/chat/ws?token=你的JWT
```

### 7.3 快速联调流程

1. 用账号A登录，搜索并添加账号B为好友。
2. 用账号B登录，同意好友申请。
3. 任一方打开“聊天”页面，选择会话发送文本消息。
4. 另一方在线时应实时收到消息（无需刷新）。
5. 发送文件消息后，接收方可在聊天气泡中下载文件。

### 7.4 聊天相关排查

- 消息不实时：确认浏览器控制台 WebSocket 连接地址为 `/api/chat/ws` 且状态为 `open`。
- 好友申请失败：确认两账号都为 `status=1` 且不是自己加自己。
- 文件消息下载失败：确认分享未过期（默认 24 小时）且接收方为被授权用户。
